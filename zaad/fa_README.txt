ZAAD – وب‌سایت کاتالوگی مبتنی بر Django
=========================================

معرفی
------
ZAAD یک وب‌سایت کاتالوگی است که با فریم‌ورک Django توسعه داده شده است.
این نسخه صرفاً بر نمایش محصولات و مدیریت روزانه از طریق پنل ادمین تمرکز دارد.
در حال حاضر هیچ سیستم سبد خرید، احراز هویت کاربر یا پرداخت آنلاین در سایت فعال نیست.
پنل مدیریت Django تنها ابزار مدیریت محتوا محسوب می‌شود.

هدف این پروژه:
- نمایش محصولات دسته‌بندی‌شده (گل، بیکری، هدیه)
- ساختار بهینه برای موتورهای جستجو (SEO)
- پشتیبانی از حالت تیره و روشن
- قابلیت استقرار آسان
- قابلیت توسعه در فازهای بعدی (سبد خرید، ورود کاربر، پرداخت)


تکنولوژی‌های استفاده‌شده
-------------------------
- Python (تست‌شده با نسخه 3.12 به بالا)
- Django (تست‌شده با نسخه 5.x)
- SQLite (برای توسعه)
- PostgreSQL (پیشنهادی برای محیط Production)
- Bootstrap (پایه طراحی رابط کاربری)
- سیستم طراحی CSS اختصاصی

امکانات اصلی
-------------
1. نمایش کاتالوگ
   - مدل پایه Product
   - فیلتر بر اساس دسته‌بندی
   - صفحه جزئیات محصول
   - آدرس‌های بهینه‌شده برای SEO

2. فرآیند مدیریت
   - افزودن و ویرایش روزانه محصولات توسط ادمین
   - بارگذاری تصویر از طریق پنل Django
   - مدیریت ساده محتوا
   - بدون ویرایشگر فرانت‌اند

3. سئو (SEO)
   - داده‌های ساختاریافته JSON-LD
   - آدرس Canonical
   - پشتیبانی از Sitemap
   - فایل robots.txt آماده
   - اسلاگ‌های تمیز و خوانا

4. رابط کاربری (UI/UX)
   - طراحی مینیمال و پریمیوم
   - سوییچ حالت تیره / روشن
   - واکنش‌گرا (Responsive)
   - هدرهای بصری بهینه


طراحی پایگاه داده
------------------
مدل پایه:
- Product (مدل پایه برای اشتراک منطق)

مدل‌های مشتق‌شده:
- Flower
- BakeryItem
- GiftItem

هر محصول شامل:
- title
- slug
- description
- price (در نسخه فعلی اختیاری)
- image
- created_at
- updated_at
- is_active


متغیرهای محیطی (Environment Variables)
---------------------------------------
الزامی:
- SECRET_KEY
- DEBUG
- ALLOWED_HOSTS

اختیاری (برندینگ / آنالیتیکس):
- GOOGLE_ANALYTICS_ID
- SITE_NAME
- CONTACT_EMAIL

نمونه فایل .env:

SECRET_KEY=your-secret-key
DEBUG=True
ALLOWED_HOSTS=127.0.0.1,localhost
SITE_NAME=ZAAD
CONTACT_EMAIL=info@example.com


فایل‌های Static و Media
------------------------
در محیط توسعه:
- STATIC_URL تنظیم شده است
- فایل‌های Media به‌صورت محلی ذخیره می‌شوند

در محیط Production:
- تعریف STATIC_ROOT الزامی است
- اجرای دستور:
  python manage.py collectstatic
- استفاده از فضای ذخیره‌سازی مناسب برای Media (پیشنهاد: فضای ابری)


نکات استقرار (Deployment)
--------------------------
چک‌لیست ساده Production:
1. تنظیم DEBUG=False
2. پیکربندی PostgreSQL
3. تنظیم ALLOWED_HOSTS
4. تعریف متغیرهای محیطی
5. اجرای migrate
6. اجرای collectstatic
7. پیکربندی وب‌سرور (Gunicorn + Nginx یا استفاده از PaaS)

پلتفرم‌های پیشنهادی:
- Render
- Liara
- Railway
- VPS اختصاصی
